# Исправление календаря и навигации

**Дата:** 06.06.2026

## Проблемы

1. ❌ **Блоки постов перекрывают друг друга** в календаре
2. ❌ **Пост не перемещается в новую дату** при изменении даты в модальном окне
3. ❌ **Нет возврата в календарь** после сохранения поста в редакторе

## Решения

### 1. Исправлена верстка календаря

**Файл:** `src/styles/main.css`

#### Проблема
`.calendar-day` имел `aspect-ratio: 1` (квадратная форма), но когда постов больше 2-3, они выходили за границы ячейки и перекрывали соседние даты.

#### Решение

```css
.calendar-day {
  aspect-ratio: 1;
  padding: 0.5rem;
  border: 1px solid var(--color-border);
  border-radius: var(--radius);
  position: relative;
  min-height: 80px;
  /* ✅ ДОБАВЛЕНО */
  display: flex;
  flex-direction: column;
  overflow-y: auto;      /* Скролл по вертикали */
  overflow-x: hidden;    /* Скрыть горизонтальный скролл */
}
```

**Преимущества:**
- ✅ Посты теперь скроллятся внутри ячейки
- ✅ Не перекрывают соседние даты
- ✅ Сохраняется квадратная форма ячеек

---

### 2. Фиксированный номер дня

**Проблема:** При скролле постов номер дня уезжал вверх и терялся из вида.

**Решение:**

```css
.day-number {
  font-size: 0.875rem;
  font-weight: 600;
  /* ✅ ДОБАВЛЕНО */
  position: sticky;      /* Остается вверху при скролле */
  top: 0;
  background: inherit;   /* Наследует фон родителя */
  z-index: 1;
  padding-bottom: 0.25rem;
  flex-shrink: 0;        /* Не сжимается */
}
```

**Результат:** Номер дня всегда виден, даже если скроллить посты.

.day-posts-mini {

### 6. Возврат в календарь после сохранения

**Файл:** `src/pages/PostEditor.tsx`

#### Проблема
После нажатия "Сохранить" показывался alert, но пользователь оставался в редакторе.

#### Решение

```typescript
const handleSave = () => {
  if (id) {
    updatePost(id, formData);
    alert('Пост сохранен');
    navigate('/calendar');  // ✅ ДОБАВЛЕНО - возврат в календарь
  }
};
```

**Результат:** 
- ✅ После сохранения автоматический возврат в календарь
- ✅ Пользователь сразу видит обновленный пост

### 7. Закрытие модального окна после изменения даты

**Файл:** `src/components/PostModal.tsx`

#### Проблема
После изменения даты/времени модальное окно оставалось открытым, и пост не перемещался визуально.

#### Решение

```typescript
const handleDateChange = () => {
  updatePost(post.id, {
    date: newDate,
    time: newTime,
  });
  setShowDatePicker(false);
  alert('Дата и время обновлены');
  onClose(); // ✅ ДОБАВЛЕНО - закрываем модалку
};
```

**Результат:** 
- ✅ Модальное окно закрывается после изменения даты
- ✅ Календарь перерендеривается
- ✅ Пост отображается в новой дате

## Технические детали

### Flexbox структура ячейки:

```
.calendar-day (flex container, column)
├── .day-number (sticky, flex-shrink: 0)
└── .day-posts-mini (flex: 1, overflow: auto)
    ├── .post-mini-card
    ├── .post-mini-card
    ├── .post-mini-card
    └── ... (скроллится если не влезает)
```

### Ключевые CSS свойства:

- `display: flex` + `flex-direction: column` — вертикальная раскладка
- `overflow-y: auto` — скролл по вертикали
- `position: sticky` — закрепление номера дня
- `flex: 1` — занятие оставшегося места
- `flex-shrink: 0` — запрет сжатия карточек
- `min-height: 0` — разрешение flex-элементу уменьшаться
