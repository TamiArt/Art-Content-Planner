Ты профессиональный senior fullstack-разработчик и архитектор PWA-приложений. Твоя задача — спроектировать и написать чистое, модульное, расширяемое браузерное приложение без сервера и без платных API. Пиши код по принципам: простая архитектура, переиспользуемые компоненты, разделение логики по модулям, минимум дублирования, понятные имена файлов и функций, легкая поддержка проекта.

Нужно создать PWA-приложение “Art Content Planner” — личный контент-оператор для художника, интерьерного скетчера и автора картин маслом. Приложение должно за 10 минут генерировать контент-план на месяц, автоматически распределять публикации по дням и времени, создавать промпты для любого ИИ, вести календарь, хранить идеи, картины, услуги, офферы, аналитику и сохранять все данные в локальный JSON-файл.

Технологии:
- React + TypeScript
- Vite
- CSS Modules или обычный CSS без тяжелых UI-библиотек
- Local state + localStorage для временного хранения
- Импорт/экспорт одного JSON-файла для постоянного хранения данных
- PWA: manifest.json, service worker, offline-ready
- Без backend, без базы данных, без регистрации, без платных API

Главные разделы приложения:
1. Dashboard — обзор месяца, ближайшие публикации, статусы, краткая аналитика.
2. Month Generator — генерация контент-плана на месяц.
3. Calendar — календарь публикаций.
4. Post Cards — карточки постов.
5. Prompt Builder — генератор промптов для любого ИИ.
6. Idea Bank — банк идей.
7. Paintings — мои картины.
8. Services — мои услуги.
9. Offers — офферы.
10. SEO / LSI — ключевые слова и хэштеги.
11. Analytics — ручной ввод и анализ результатов публикаций.
12. Settings — настройки ниши, расписания, платформ, стиля, JSON-импорт/экспорт.

Основная логика генерации:
Пользователь выбирает месяц, частоту публикаций, дни недели, время публикаций, платформы и цели. По умолчанию: 3 поста в неделю. Например:
- понедельник — охват;
- среда — доверие;
- пятница — продажа или заявка.

Генератор должен работать не случайно, а по правилам:
день публикации → цель → этап воронки → тема → формат → шаблон → промпт.

Контент-воронка:
1. Привлечение — ошибки, процесс, до/после, быстрые скетчи.
2. Удержание — польза скетча, цвет, композиция, интерьерные решения.
3. Доверие — процесс, подход, отзывы, детали работы.
4. Заявка/продажа — услуги, картины, офферы, как заказать.

Prompt Builder:
Приложение не должно обращаться к AI API. Оно должно собирать сильный промпт и давать кнопку “Скопировать промпт”. Пользователь сможет вставить этот промпт в ChatGPT, Claude, Gemini или любой другой ИИ.

Промпт должен учитывать стиль пользователя:
- живой;
- экспертный;
- мягкий;
- визуальный;
- без банальностей;
- без агрессивных продаж;
- без слишком длинных текстов;
- с акцентом на интерьер, свет, цвет, глубину, настроение и пространство.

В промпте нужно просить ИИ создать:
1. 5 вариантов хука.
2. Текст или сценарий.
3. Визуальный сценарий по кадрам.
4. CTA.
5. Ровно 5 хэштегов.
6. Короткую версию описания.
7. Встроить LSI-ключи естественно, без SEO-спама.

SEO / LSI-модуль:
Создать кластеры ключей:
1. Интерьерные скетчи.
2. Картины маслом.
3. Картина в интерьере.
4. Процесс рисования.
5. Ошибки и обучение.
6. Продажа и заявки.

Для каждого поста приложение должно подбирать:
- главный ключ;

- ключи для первой строки;
- ключи для текста на экране;
- ключи для озвучки;
- ключи для описания;
- ровно 5 хэштегов.
