## Логика работы после исправления

### До исправления:
```typescript
const addPosts = (posts: Post[]) => {
  const newData = { ...data, posts: [...data.posts, ...posts] };
  // ❌ data.posts может быть устаревшим из замыкания!
  setData(newData);
  saveAppData(newData);
};
```

**Проблема:** `data` захватывается из замыкания и может быть устаревшим.

### После исправления:
```typescript
const addPosts = (posts: Post[]) => {
  setData((prevData) => {
    // ✅ prevData всегда актуальное - гарантия React
    const newData = { ...prevData, posts: [...prevData.posts, ...posts] };
    saveAppData(newData);
    return newData;
  });
};
```

**Решение:** React гарантирует что `prevData` - это последнее актуальное состояние.
